# 同事使用方式

如果同事打不开你电脑上的局域网地址，可以改成“每个人本地运行一份看板”。

## 怎么发给同事

把整个 `AI比赛` 文件夹压缩成 zip，发给同事，或放到公司共享盘。

同事解压后，双击：

```text
start-dashboard.bat
```

正常情况下浏览器会自动打开：

```text
http://127.0.0.1:18787
```

命令行窗口不要关。关掉窗口后，看板服务也会停止。

## 如果浏览器显示“拒绝连接”

这通常不是 GitHub 的问题，而是本机看板服务没有启动成功。按顺序检查：

1. 确认已经先解压 ZIP，不要在压缩包里直接双击。
2. 确认双击的是 `start-dashboard.bat`。
3. 确认浏览器地址是 `http://127.0.0.1:18787`，不是 `8787` 或 `8000`。
4. 确认黑色启动窗口还开着。窗口关掉后，看板也会停。
5. 如果黑色窗口提示没有 Python，先安装 Python 3.10 或以上版本，安装时勾选 `Add Python to PATH`。

## 如果提示没有 Python

这台电脑需要先安装 Python 3.10 或以上版本。安装时勾选 `Add Python to PATH`，安装后重新双击 `start-dashboard.bat`。

## 这种方式的特点

- 不需要连你的电脑。
- 不需要改你电脑防火墙。
- 每个人都在自己电脑本地使用。
- 每个人搜索出来的数据存在自己的电脑里，不是多人共用同一份记录。

## 如果要所有人共用同一个网址

需要把看板部署到一台大家都能访问的公司内网机器、云主机或带登录保护的服务入口上。这种方式更适合长期团队使用。
